## 任务：跨书概念关联

发现读者已阅读的多本书之间的概念联系，帮助构建更完整的知识网络。

### 执行步骤

1. **扫描所有书籍**
   - 读取 `books/*/reading-progress.md` 获取所有书的已掌握概念
   - 读取 `books/*/book-info.md` 了解各书的类别和主题

2. **分析概念关联**

   寻找以下类型的连接：

   - **相似概念**：不同作者用不同方式描述相同道理
   - **互补概念**：不同书的概念互相强化、补充
   - **矛盾观点**：不同作者持相反立场，值得深入思考
   - **递进关系**：一本书的概念是另一本书的基础或延伸

3. **如果用户指定了概念（$ARGUMENTS）**
   - 聚焦分析该概念与其他书中概念的关联
   - 给出具体的对比和分析

4. **输出关联发现**

   输出格式示例：

   ```
   🔗 跨书概念关联

   【相似概念】
   • 金字塔原理的"结论先行" ↔ [另一本书]的"电梯演讲法则"
     → 两者都强调先传达核心信息，但切入角度不同...

   【互补概念】
   • 金字塔原理的"MECE分组" + [另一本书]的"系统思考"
     → MECE 帮你拆分问题，系统思考帮你看到整体...
   ```

5. **更新进度文件**
   - 将发现的关联写入各书的 reading-progress.md "跨书连接"部分

### 前提条件

- 至少需要 2 本书有已掌握的概念才能进行关联分析
- 如果只有 1 本书，提示用户开始阅读更多书籍后再使用此功能

### 语言要求

- 以中文输出所有内容
