## 任务：每日书籍深度推荐

为用户推荐一本高质量的书籍，类别限定为**管理与领导力、沟通与表达、心理与行为、哲学与人文、思维与认知**。不只是推荐，而是像一位深度阅读者一样分享这本书的精华，并用苏格拉底提问和费曼学习法引导读者思考。

### 执行步骤

1. **读取已推荐记录**
   - 读取 `recommendations/history.md`，获取已经推荐过的书名列表
   - 确保本次推荐不与历史记录重复

2. **读取用户已在读的书籍**
   - 扫描 `books/*/book-info.md`，获取用户已在读和读完的书
   - 避免推荐用户已经在读的书

3. **选择推荐书籍**
   - 从以下类别中轮换推荐：
     - **管理与领导力**：领导力、团队管理、战略规划、组织行为、变革管理
     - **沟通与表达**：表达技巧、演讲、写作、谈判、说服力、非暴力沟通
     - **心理与行为**：认知心理、行为经济学、情绪管理、社会心理、积极心理学
     - **哲学与人文**：人生哲学、伦理学、存在主义、东方哲学、美学、逻辑学
     - **思维与认知**：系统思考、思维模型、决策科学、批判性思维、复杂性理论、第二曲线、长期主义
   - 优先推荐经典名著和高口碑书籍
   - 用 WebSearch 搜索书籍的详细章节内容、核心观点、经典案例

4. **输出推荐内容**（严格按以下结构，每个部分都不能省略）

   **内容必须包含以下 9 个部分：**

   ---

   **Part 1: 基本信息**

   ```
   推荐：每日书籍推荐 — YYYY-MM-DD

   《书名》（English Title）
   作者：XXX
   类别：管理与领导力 / 沟通与表达 / 心理与行为 / 哲学与人文 / 思维与认知

   推荐理由：
   [2-3 句话说明为什么推荐这本书，适合什么样的读者，能解决什么问题]
   ```

   ---

   **Part 2: 章节详解**（每一章都要列出，不能只写大纲标题）

   ```
   章节详解：

   第一部分：[部分标题]

   第1章：[章节标题]
   本章主要内容：[3-5 句话概括本章核心内容，不是一句话标题，而是让没读过的人能理解这章在讲什么]

   第2章：[章节标题]
   本章主要内容：[同上]

   ...（每一章都要写）
   ```

   ---

   **Part 3: 核心观点**（至少 5-8 个，要有深度）

   ```
   核心观点：

   1. [观点标题] — [2-3 句详细解释，不是一句话概括，而是让人真正理解这个观点的含义和价值]

   2. [观点标题] — [同上]

   ...（至少 5 个，尽量 8 个）
   ```

   ---

   **Part 4: 经典案例**（至少 5 个，越多越好，帮助读者通过故事理解抽象概念）

   ```
   经典案例：

   案例1：[案例标题]
   背景：[1-2 句交代场景和人物]
   过程：[2-3 句描述发生了什么，当事人怎么做的]
   结果：[1-2 句说明结局和效果]
   启示：[1 句话点明这个案例印证了书中的哪个核心观点]

   案例2：[案例标题]
   [同上结构]

   ...（至少 5 个案例，覆盖书中不同章节和观点。案例来源可以是：
   - 书中原文的案例和故事
   - 作者亲身经历的真实事件
   - 书中引用的研究实验
   - 与书中理论相关的知名企业/人物案例
   每个案例必须具体、有细节，不能泛泛而谈）
   ```

   ---

   **Part 5: 深度读后感**（站在深度阅读者角度）

   ```
   深度读后感：

   [以第一人称"我"的视角，写 150-200 字的读后感。包括：
   - 这本书最打动我的是什么
   - 它改变了我哪些认知
   - 我在实际工作/生活中如何应用了书中的方法
   - 这本书的局限性或我不完全认同的地方
   语气要真诚、有个人见解，不要写成书评模板]
   ```

   ---

   **Part 6: 苏格拉底式思考题**（引导读者深入思考）

   ```
   思考题（读前或读后都可以想一想）：

   1. [一个开放式问题，没有标准答案，引导读者联系自己的经历思考]
   2. [一个挑战性问题，让读者反思自己的现有认知或习惯]
   3. [一个应用性问题，让读者想象如何把书中方法用到具体场景]
   ```

   ---

   **Part 7: 费曼检验**（用最简单的语言总结）

   ```
   费曼检验 — 一句话说清这本书：

   [用一句通俗易懂的话总结整本书的核心，就像解释给一个完全不了解这个领域的朋友听。不用术语，不用抽象概念。]

   如果要解释给小学生听：
   [用更简单的类比或故事，让一个 10 岁的孩子也能大概理解这本书在说什么]
   ```

   ---

   **Part 8: 读后输出行动**（读完不输出等于没读，给出具体可执行的输出任务）

   ```
   读后输出行动（读完这本书后，试着完成以下练习）：

   输出1：[具体的写作/实践任务]
   做法：[1-2 句说明怎么做]
   预计用时：[X 分钟]

   输出2：[具体的写作/实践任务]
   做法：[同上]
   预计用时：[X 分钟]

   输出3：[具体的写作/实践任务]
   做法：[同上]
   预计用时：[X 分钟]

   [要求：
   - 输出任务必须具体可执行，不能是"思考一下"这种模糊的要求
   - 类型多样化：可以是写一段文字、画一张思维导图、做一次实验、跟人做一次练习、改写一段旧文档等
   - 至少一个任务是"教别人"——用自己的话给朋友/同事讲一遍书中的核心概念
   - 至少一个任务是"用到真实场景"——在下一次开会/写邮件/沟通中实际应用
   - 标注预计用时，让读者觉得"这个我可以做到"而不是望而却步]
   ```

   ---

   **Part 9: 跨书关联与行动建议**

   ```
   与已读书籍的关联：
   [如果与用户已读的书有概念关联，具体指出相似、互补或矛盾之处]

   读完这本书后，推荐接着读：
   [推荐 1-2 本相关书籍，说明为什么形成阅读链]

   想开始读这本书？输入 /new-book [书名]
   ```

5. **更新推荐记录**
   - 将本次推荐追加到 `recommendations/history.md`
   - 记录：日期、书名、作者、类别

### 质量要求

- **章节详解必须覆盖每一章**，不能只列标题
- **核心观点要有解释**，不是一句话清单
- **案例至少 5 个**，必须有背景、过程、结果、启示，覆盖书中不同章节
- **读后感要有个人见解**，不是通用书评
- **思考题要有启发性**，不是简单的知识问答
- **费曼检验要通俗**，真正做到外行能懂
- **读后输出必须可执行**，有具体做法和预计用时，至少包含"教别人"和"真实场景应用"

### 语言要求

- 以中文输出所有内容
- 英文书名附原文
- 消息正文中必须包含"推荐"二字（钉钉关键词要求）
