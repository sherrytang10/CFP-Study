## 任务：每日书籍深度推荐

为用户推荐一本高质量的书籍，类别限定为**管理与领导力、沟通与表达、心理与行为、哲学与人文、思维与认知**。不只是推荐，而是像一位深度阅读者一样分享这本书的精华，并用苏格拉底提问和费曼学习法引导读者思考。

### 执行步骤

1. **读取已推荐记录**
   - 读取 `recommendations/history.md`，获取已经推荐过的书名列表
   - 确保本次推荐不与历史记录重复

2. **读取用户已在读的书籍**
   - 扫描 `books/*/book-info.md`，获取用户已在读和读完的书
   - 避免推荐用户已经在读的书

3. **选择推荐书籍**
   - 从以下类别中轮换推荐：
     - **管理与领导力**：领导力、团队管理、战略规划、组织行为、变革管理
     - **沟通与表达**：表达技巧、演讲、写作、谈判、说服力、非暴力沟通
     - **心理与行为**：认知心理、行为经济学、情绪管理、社会心理、积极心理学
     - **哲学与人文**：人生哲学、伦理学、存在主义、东方哲学、美学、逻辑学
     - **思维与认知**：系统思考、思维模型、决策科学、批判性思维、复杂性理论、第二曲线、长期主义
   - 优先推荐经典名著和高口碑书籍
   - 用 WebSearch 搜索书籍的详细章节内容、核心观点、经典案例

4. **输出推荐内容**

   **⚠️ 致命红线（违反任何一条 = 任务失败）：**

   **❌ 错误示范（绝对不能写入文件的内容）：**
   ```
   质量检测通过：
   - **总行数**：211 行 (≥ 200) ✅
   - **9 个必需二级标题**：全部存在 ✅
   ...
   今日推荐完成：**《XXX》**
   ```
   ↑ 这是对话汇报消息，不是文件内容。如果你的 Write content 长这样，你搞错了！

   **✅ 正确示范（文件内容必须以此开头）：**
   ```
   # 每日书籍推荐 — 2026-04-14

   ## 《书名》（English Title）

   **原书名**：...
   **作者**：...
   ```
   ↑ 文件的第一行必须是 `# 每日书籍推荐 — YYYY-MM-DD`。如果第一行不是 `#` 开头，立即停止——你写错了。

   **三条硬规则：**
   1. Write 工具的 content 参数 = 9 大板块的完整正文（200+ 行），不是摘要、不是检测报告、不是完成通知
   2. 对话中给用户的汇报消息**永远不写入文件**——汇报是说给人听的，文件是留给读者看的
   3. 调用 Write 之前，检查你的 content 参数的第一行是否以 `#` 开头——如果不是，你正在写汇报消息而不是推荐正文

   **生成前必做**：先 Read `recommendations/坚毅.md`，学习其格式和内容深度，然后严格照此格式生成。

   **文件结构规范（9 个 `##` 二级标题，缺一不可，板块之间用 `---` 分隔）：**

   `# 每日书籍推荐 — YYYY-MM-DD` — 一级标题

   `## 《书名》（English Title）` — 含 **原书名**、**作者**、**类别**（五选一）、**适读人群**

   `## 推荐理由` — 2-3段，200-300字，说明为什么推荐、适合谁、解决什么问题

   `## 章节详解` — 用 `### 第X章：标题` 逐章展开，每章写3-5句话概括核心内容（不是一句话标题），全书每一章都必须写到

   `## 核心观点` — 至少5个（尽量8个），格式：`**1. 观点标题** — 2-3句详细解释`，要让没读过的人能理解这个观点的含义和价值

   `## 经典案例` — 至少5个，格式：`### 案例N：标题` 下含 **背景**（1-2句）/ **过程**（2-3句）/ **结果**（1-2句）/ **启示**（1句）四要素，必须具体有细节，覆盖不同章节。来源：书中原文案例、作者经历、研究实验、相关企业/人物案例

   `## 深度读后感` — 第一人称"我"，150-200字。必须包含：最打动我的是什么、改变了哪些认知、如何应用、局限性或不完全认同的地方。语气真诚有个人见解

   `## 思考题` — 3个编号问题：1个开放式（联系自身经历）、1个挑战性（反思现有认知）、1个应用性（具体场景应用）

   `## 费曼检验` — 含 `**一句话说清这本书**`（通俗无术语）+ `**解释给小学生听**`（类比或故事，10岁孩子能懂）

   `## 读后输出行动` — 至少3个，格式：`### 行动N：任务名` 下含 **做法**（1-2句）+ **预计用时**（X分钟）。要求：具体可执行、类型多样化、至少一个"教别人"、至少一个"真实场景应用"

   `## 跨书关联` — 与已读/已推荐书籍的概念关联（相似/互补/矛盾）+ **读完后推荐接着读**（1-2本）+ 结尾：`> 想开始读这本书？输入 /new-book 书名`

5. ~~**更新推荐记录**~~（此步骤已由 `scripts/update-history.py` 本地自动完成，Claude 无需操作 history.md）

6. **生成后质量检测（必须执行，不可跳过）**

   **⚠️ 关键防错规则（写入文件后、汇报前必须逐条核实）：**
   - 用 Read 工具读取刚写入的文件的**前 5 行**——如果第一行不是以 `#` 开头，说明写入了汇报消息而非推荐正文，必须立即用 Write 重新写入正确内容
   - 严禁凭记忆声称行数。必须用 Bash `wc -l` 或 Read 工具获取实际行数
   - 严禁凭记忆声称板块齐全。必须用 Grep `^## ` 扫描实际存在的二级标题

   **检测步骤：**
   1. 用 Read 工具读取文件前 5 行，确认第一行是 `# 每日书籍推荐 — YYYY-MM-DD`（如果不是 → 文件内容错误，用 Write 重写）
   2. 用 `wc -l` 核实实际行数 ≥ 200（少于 200 行 → 内容缺失，用 Write 重写）
   3. 用 Grep `^## ` 扫描文件，逐一确认以下 9 个标题全部存在：
      `## 推荐理由` · `## 章节详解` · `## 核心观点` · `## 经典案例` · `## 深度读后感` · `## 思考题` · `## 费曼检验` · `## 读后输出行动` · `## 跨书关联`
   4. 抽检内容深度：章节详解是否逐章展开（非一句话标题）、案例是否 ≥ 5 个且有四要素、观点是否 ≥ 5 个且有解释
   5. 如有不足，立即用 Edit 工具补全至标准深度，补全后再次 Read 验证
   6. **全部通过后**，在对话中向用户汇报完成情况（汇报消息只出现在对话里，绝不写入文件）

### 质量要求

- **章节详解必须覆盖每一章**，不能只列标题
- **核心观点要有解释**，不是一句话清单
- **案例至少 5 个**，必须有背景、过程、结果、启示，覆盖书中不同章节
- **读后感要有个人见解**，不是通用书评
- **思考题要有启发性**，不是简单的知识问答
- **费曼检验要通俗**，真正做到外行能懂
- **读后输出必须可执行**，有具体做法和预计用时，至少包含"教别人"和"真实场景应用"

### 语言要求

- 以中文输出所有内容
- 英文书名附原文
- 消息正文中必须包含"推荐"二字（钉钉关键词要求）
